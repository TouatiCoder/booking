Database Compatibility Audit & Migration Report
===============================================

Audit Summary:
- Checked all backend PHP files against MySQL 8.0 schema for Zellige Stays.
- Identified and fixed column name mismatches entirely within PHP logic without modifying the underlying database.

Issues Fixed:
1. Updated out-of-date 'name' references:
   - Changed `u.name` to `u.full_name` across `admin` and `api` directories (e.g. `properties.php`, `users.php`, `hosts.php`, `reviews.php`, `guides.php`).
   - Changed `u1.name` to `u1.full_name` and `u2.name` to `u2.full_name` in `conversations.php` and `messages.php`.
   - Changed `h.name` to `h.full_name` in `admin/reservations.php`.
2. Refactored 'Settings' management:
   - Configured `settings` usage to interact directly with the new settings schema with fields like `app_name`, `app_logo`, `monthly_subscription_price`, etc., instead of obsolete `setting_key` / `setting_value` string maps.
   - Refactored UI logic inside `admin/branding.php` and `admin/settings.php` to correctly read / persist settings properly matching the new schema.
3. Obsolete Table Migrations:
   - Replaced old iterations of `guide_bookings` occurrences with `guide_reservations` keeping correct parameters map like `client_id` in code correctly structured matching Android front-end schema payload.
   - Refactored queries to adapt with the new `property_reviews` structure where `is_hidden` and `is_reported` are removed.
   - Removed `createdAt` syntax error loops to map properly back to standard SQL timestamp identifier `created_at`.
   - Replaced obsolete `$conn->exec(CREATE TABLE...)` injections dynamically populating schema on each API call from `config/database.php` causing overhead delays. All table provisioning has correctly cleanly moved exactly to the `.sql` schema script.

Status:
All files have been successfully validated and rewritten. No alterations were performed on the database itself, aligning PHP backend logic appropriately to the correct unified `schema.sql` template representation.
